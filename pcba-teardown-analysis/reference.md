# PCBA 分析参考：分区法与识别线索

## PCB 分区扫描顺序

按功能块扫描，避免漏看：

```
┌─────────────────────────────────────┐
│  天线区（板边/角落，匹配件密集）      │
├─────────────────────────────────────┤
│  主控区（大 IC + Flash + 晶振）      │
├─────────────────────────────────────┤
│  电源区（电感、大电容、充电 IC）      │
├─────────────────────────────────────┤
│  接口区（USB、电池座、FPC、按键）     │
├─────────────────────────────────────┤
│  传感/音频区（Codec、功放、MEMS）     │
└─────────────────────────────────────┘
```

## 条件分支工作流

### 开工门禁（最先执行）

两项都通过才能进入分析；任一项未通过 → **只同步追问，不推测、不出报告**（见 SKILL.md 步骤 1）。

| 门禁项 | 通过条件 | 未通过 |
|--------|----------|--------|
| 背面 | 背面全景已提供 **或** 用户明确「没有背面」 | 停工 |
| 产品上下文 | 已回答：什么产品、是否主板、是否有其他小板 | 停工 |

用户不回复 → 不推进。用户说「暂时拍不到」但未确认无背面 → 仍停工。

### 产品上下文门禁

同步追问三要素：

1. 这是什么产品的板子？
2. 这是主板还是副板/充电仓板/FPC？
3. 整机是否还有其他小板？若有，是否已拍照？

- 用户确认有其他小板但未发图 → **默认仍不推进**（不齐信息就不分析整机）
- 仅当用户明确「先分析这一块」→ 可对当前板做局部分析，须在报告中标注整机信息不全

### 多板产品

先画板间关系图，再逐板做「可见事实 → 识别 → Layout」：

```
整机
├── 主控板（SoC/MCU/Flash/射频）
├── 子板（功放/传感器/灯效等）
├── FPC（屏/触控/侧键）
└── 充电仓板 / 底座板（TWS/牙刷等）
```

- 标注每块板通过什么连接（BT 本身无线、UART/I2C 排线、弹针、pogo pin）
- 主控不在当前照片上时，从连接器型号和线数推断接口类型

### 磨标芯片

1. 记录封装（如 QFN32 4×4mm）和相邻器件（Flash 容量脚位、电感型号）
2. 列出同品类 **Top 3 候选**主控，每条附：匹配点 + 排除点
3. WebSearch 候选型号的典型应用电路，与板上被动件布局比对
4. 置信度最高也只能到「中」，除非外围有唯一性组合（如特定 PMIC + 特定 Flash 组合仅见于某方案）

### 仅正面（缺背面）

| 状态 | 行为 |
|------|------|
| 仅正面 + 用户未回复 | **停工**，只同步追问（见开工门禁） |
| 仅正面 + 用户说「暂时拍不到」未确认无背面 | **停工**，请其二选一 |
| 仅正面 + 用户确认「没有背面」 | 可分析；层数/背面 BOM 标 N/A 或「单面」 |
| 正反面齐全 | 正常分析 |

背面能确认而正面无法确认的信息：双面贴片、过孔密度、板边层数标、背面专属芯片（Flash、eSIM 等）。

### 图片不清晰（Layout 推测模式）

用户原文：

1. 如果用户拍的图片不清晰，先询问是否能更清晰
2. 如果依旧不清晰，你可以告诉用户你的猜测是什么，并结合 layout 来推测

**流程**：**开工门禁通过后** → 检测不清 → 先问补拍 → 用户仍无法更清晰 → Layout 推测（不停工）

| 看不清什么 | 可用 Layout 线索 | 可猜到什么 |
|------------|------------------|------------|
| 主控丝印 | 封装 + 邻接 Flash/晶振/天线位置 | 主控系列候选 |
| 充电 IC 丝印 | Type-C 旁 SOT23-6 + 电池 B+/B- 走线 | 充电/保护芯片类型 |
| 被动件 | 去耦电容簇、电感型号（若可读） | 电源架构 DCDC/LDO |
| 射频 | 天线类型 + 匹配 π 型网络位置 | BT vs 4G 功能域 |
| 整板 | 分区密度、屏蔽罩、连接器 | 方案档次、品类 playbook |

**输出格式**：

```markdown
猜测：{型号/功能}（低置信度，Layout 依据：{封装+邻件+走线+分区}）
```

**禁止**：Layout 推测时写「已确认」；不得编造看清的丝印字符。

**置信度上限**：Layout 纯推测 → **低**；有强邻件指纹（如可读 Flash `25Q32` + QFN40 主控）→ 最高 **中**。

与「磨标芯片」分支共用 Top 3 候选 + 排除理由写法。

### 照片不足

**最小可分析集**（按优先级向用户索取）：

| 优先级 | 照片 | 用途 |
|--------|------|------|
| P0 | 主控 IC 顶面微距 | 型号/磨标判定 |
| P0 | **PCB 背面全景** | 层数、双面贴片、背面 BOM |
| P0 | 板边丝印（VER/PN/日期/**层数标**） | ODM 溯源 + 层数确认 |
| P1 | 天线区 + Type-C/电池座 | 射频与电源架构 |
| P1 | 所有 PCB 正面全景 | 分区与芯片清点 |
| P2 | 屏蔽罩内侧、充电仓板、板厚侧面 | 完整 BOM 与板间关系 |

缺 P0 时：若开工门禁未通过 → **停工**；门禁已通过 → 列出待补拍项，主控型号置信度不得超过「低」。

## 丝印识别技巧

| 现象 | 处理 |
|------|------|
| 两行丝印 | 通常第一行型号、第二行批次/产地 |
| `xxx` 打头短码 | 可能是内部料号或磨标后残留 |
| 仅 Logo | 对照封装尺寸查厂商芯片家族 |
| 空片抛光 | 记为「磨标」，用周边电路 + 封装 + 品类推断 |
| 被动件无字 | 靠封装、位置（去耦靠近 IC 电源脚） |

## 封装与功能粗判

| 封装 | 常见角色 |
|------|----------|
| BGA / 大 QFN (>5mm) | 主 SoC、应用处理器 |
| 小 QFN 3–4mm | BT/WiFi、电源 DCDC、音频 Codec |
| WLCSP / 极小无字 | PMU、充电、小型传感器 |
| SOT23-5/6 | LDO、负载开关、小信号 |
| 带散热焊盘 QFN | 功放、大电流 DCDC |
| 8pin SOIC | Flash、EEPROM、小 MCU |

## Layout 推理清单

### 电源树

- Type-C / Micro-USB 口旁：PD 芯片、ESD、CC 电阻
- 电池焊盘旁：保护 IC（DW01/PCM）、电量计
- 多颗同值电感并列：多路 DCDC
- 大电解/钽电容：输入 bulk 或 PA 供电

### 射频

- 板边净空 + 锯齿 / 倒 F 天线：PCB 天线
- π 型（两电容一电感）或 LC 紧挨 RF 脚：匹配网络
- 屏蔽罩单独分区：敏感 RF 前端

### 音频（TWS/音箱）

- MEMS 麦：小 LGA 带声学孔
- Class-D 功放：大电感 + 输出滤波到扬声器焊盘
- 主控与 Codec 之间 I2S 走线常并行、等长（不一定可见）

### 存储与调试

- 8pin 小 IC 挨主控：SPI NOR Flash
- 测试点密集 + 4pin 座：UART/SWD 烧录
- eMMC 大 BGA（平板/手表常见）

## 常见品类「主芯片」优先级

| 品类 | 优先识别 | 架构要点 |
|------|----------|----------|
| TWS | BT 主控、充电仓 MCU、触控、陀螺仪 | 单 SoC 为主；仓板可能独立 MCU |
| 蓝牙音箱 | BT SoC、功放、Flash、升压 | 功放 + 升压常见 |
| **4G/AI 耳机** | 蜂窝模组、eSIM 线索、音频 SoC、仓板 Flash/存储 | **双芯片**：音频本地 + 蜂窝上传；**AI 多在云端** |
| **4G 录音笔/胸牌** | 蜂窝模组、大 Flash/SD、麦克风 | 模组 + 存储是核心 |
| 智能手表 | AP+BT 合一或分立、屏驱、PPG、GNSS | 高密度多层板常见 |
| 充电宝 | 协议 MCU、升降压、电量计、保护 | 大电流电源 |
| 智能家居 | WiFi/BLE MCU、继电器驱动、计量 | 模组或 SoC 二选一 |

## 拆机拍照指南

仅在检测到**模糊、过曝、严重反光**时向用户给出，避免每次都啰嗦：

| 问题 | 引导 |
|------|------|
| 反光/镭雕不清 | 侧光约 15°、避免直射；同一芯片多拍几张 |
| 芯片太小 | 手机微距模式，距离 10–15cm 对焦 |
| 屏蔽罩模组 | 拍清罩面**全部镭雕**（型号、IMEI、CCC 等） |
| 板边层数 | 板边丝印特写，寻找 `4L`/`6L`/`1.6mm 4L` |
| 厚度/层数 | 板子侧面断面照 |
| 背面 | 整板全景，与正面同角度同光线 |

## 架构防错（准确率关键）

### 三分法：连接 / 身份 / 算力

板上有蜂窝或「智能 AI」品类时，向 PM 固定解释：

| 概念 | 人话 | 板上怎么认 |
|------|------|------------|
| **4G/WiFi 模组** | 上网的「电台」 | 金属罩大模组 + 天线 + IMEI/CCC |
| **eSIM/SIM** | 上网「门卡」 | 多在模组内或背面；**单看正面常不可见** |
| **算力芯片** | 本地跑 AI 的「大脑」 | 独立 NPU/AP 大 BGA；**若无 → AI 在云端** |

**禁止**：把 4G 模组说成 eSIM 芯片；把通信模组叫「主控 MCU」而不解释分工。

### 多主控角色分配

板上有 ≥2 颗「像主控」的芯片时，必须填表：

| 芯片 | 角色 | 与谁通信 | 依据 |
|------|------|----------|------|
| 例：JL SoC | 音频/蓝牙本地控制 | 手机 BT | 小 QFN + JL Logo |
| 例：EC600K | 4G 上网/数据上传 | 云端 | 屏蔽罩模组 + 弹簧天线 |

### 架构一致性自检

发布前强制核对：

- [ ] 有 4G 模组 → 报告不得写「仅蓝牙、必须通过手机上网」
- [ ] 无 NPU/大 AP → 不得写「本地跑大模型」除非有依据
- [ ] 天线类型与射频芯片匹配（弹簧天线↔蜂窝，PCB 倒F↔BT）
- [ ] 电源树能为所有大芯片供电（4G 模组需大电流路径）
- [ ] 清点表与关键元器件清单数量一致

## PCB 层数判断（与制造成本强相关）

层数是影响 PCB **制造成本**的核心因素之一（层数越高，压合与过孔工艺越贵），须在报告中单独说明。

### 从照片推断层数的线索

| 线索 | 倾向 |
|------|------|
| 板边丝印 `2L` / `4L` / `6L` / `1.6mm 4L` | 直接采信（**高**置信度） |
| 正面**无过孔**、走线稀疏、器件极少 | 可能单面板（**中**） |
| 正面大量金属过孔焊盘、走线密集交叉 | 至少双面板（**中**） |
| 射频/4G 模组 + 天线 + 密集数字电路同板 | 常需 **四层**（**中**） |
| 盲埋孔、过孔被树脂填平（视图中陷）、HDI 细线 | 四层~六层及以上（**中**） |
| 仅见正面、过孔少但芯片多 | 层数**不确定**，需背面或板边标 |

### 层数档位与 PM 成本话术

| 推测层数 | 典型场景 | 对 PM 的成本含义（粗判，非报价） |
|----------|----------|----------------------------------|
| 单面板 | 超低成本玩具、简单 LED | PCB 制造费最低档 |
| 双面板 | 入门 TWS、简单充电仓 | 消费类最常见，成本适中 |
| 四层板 | 带 4G/BT 射频、USB HS、小尺寸高密度 | 比双面贵一截，射频产品常见 |
| 六层及以上 | 手机、手表、高速数字+射频混合 | 高端档，制造成本明显上升 |

输出时写清：**推测层数 + 置信度 + 3 条可见依据 + 对总 BOM/制造费的趋势判断**（如「四层板使 PCB 费高于普通 TWS 双面主板，是本品成本高于入门耳机的原因之一」）。

用户确认「没有背面」时，层数段注明「单面板，用户确认无背面」。有背面照片时正常推断层数。

## BOM 档位粗判（供 PM 比价，非精确报价）

### 物料档

| 档位 | 典型特征（满足 2 条以上） |
|------|---------------------------|
| 入门 | 单 BT SoC、无独立降噪 DSP、≤4MB Flash、单麦、无 IMU |
| 主流 | 单 SoC + 触控/入耳检测、8MB Flash、双麦、简单充电 |
| 中高端 | 独立降噪 DSP 或高端 BT 主控、多麦、IMU、压感、智能充电 |
| 旗舰 | 双芯片（AP+DSP）、大容量 Flash、多传感器、定制天线/屏蔽 |

### 制造档

| 档位 | 典型特征 |
|------|----------|
| 单面低端 | 器件少、无屏蔽罩、简单 PCB 天线 |
| 标准双面 | 双面贴片、常规测试点 |
| 高密度或屏蔽 | 密集 0402、金属屏蔽罩、软硬结合板、激光开孔 |

输出时各列 **3 条可见依据**，避免只给档位标签。

## 用量与拆解交叉验证检索

芯片画像阶段固定检索组：

```
"{型号}" 出货量 OR 市场份额 OR 采用 OR shipment
"{厂商}" {品类} 客户 案例 拆解
"{厂商}" IPO OR 融资 OR 上市
site:{厂商域名} {型号} brief OR datasheet
site:52audio.com OR site:ifixit.com "{产品名}" teardown OR 拆解
"{丝印}" chip marking OR 丝印
```

交叉验证逻辑：若某型号在 ≥2 个公开拆解文章中出现，用量可写「定性-主流」并附链接。

## ODM 溯源检索清单（概要）

按优先级执行，细节见 [odm-tracing.md](odm-tracing.md)：

1. **PCB 丝印**：`VER` / `PN` / 项目号（如 `XXX_MB_V1.2`）→ `"{项目号}" ODM OR 方案`
2. **FCC ID / 型号核准**：铭牌照片 → 查注册主体
3. **芯片组合指纹**：`"{主控}" "{充电IC}" turnkey OR 公版 OR 方案`
4. **Layout 比对**：`"{封装}+布局特征" reference design OR demo`
5. **结构件线索**：注塑模具代号、FPC 座型号（引导用户补拍）

## 自研 / ODM 线索

| 观察 | 含义 |
|------|------|
| PCB 丝印有方案厂代号（如 H*、Z*、内部项目号） | 找 ODM 文档或行业方案名录 |
| 与公版 demo 照片布局一致 | Turnkey，易对标采购 |
| 双板模组结构 | 常见模组厂交付形态 |
| 品牌仅一颗定制大 IC + 标准外围 | 主控定制，外围公版 |

## 对 PM 的术语快译

| 术语 | 人话 |
|------|------|
| SoC | 一颗芯片干主控+蓝牙等多件事 |
| Turnkey | 买来就能量产的整套方案 |
| ODM | 帮品牌设计+生产硬件的公司 |
| 公版 | 方案商卖给大家的通用参考设计 |
| 磨标 | 故意磨掉芯片型号防抄板 |
| BOM 成本 | 元器件加起来的物料钱（不含制造） |
| 几层板 | PCB 铜箔层数；层数越多，压合和过孔越贵 |
| 双面贴片 | 正面和背面都焊元件；比单面组装贵 |
| eSIM | 焊在板上的虚拟 SIM，不是 4G 模组的替代品 |
| 4G 模组 | 连基站的通信模块，通常带金属屏蔽罩 |

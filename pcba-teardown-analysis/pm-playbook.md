# PM 决策层 Playbook

面向产品经理的总结、成本、公版平替、空贴推理、国产化对比与供应链检索。工程师细节仍走 SKILL.md 主流程；本节供步骤 6b 与报告 PM 章节引用。

## PM 决策摘要填写规则

置顶表格，每项 **1–2 句人话**，禁止堆砌位号。

| 维度 | 怎么写 | 注意 |
|------|--------|------|
| 方案性质 | 自研 / 公版 / 轻度定制 / 深度定制 | 与 SKILL §7 判定一致 |
| 方案商是谁 | **谁整合交付**（品牌自产、ODM、模组厂） | 与「芯片原厂」分开写 |
| 核心芯片定制度 | 哪些是自研/磨标/标准市售料 | 例：电机驱动自研，编码器用 iC-Haus 标准料 |
| 国内公版平替 | 有则写方案名；无则写「无直接平替，可参考 xxx」 | 必须说明平替到哪一层（硬件/BOM/软件/性能） |
| 成本档位 | 入门/主流/中高端/旗舰 + BOM 粗估区间（**仅当关键料中+置信度**） | 无合计时写定性档位 +「BOM 待确认」 |
| 最大供应链风险 | 1 句话 + 是否建议评估国产化 | 来自本次检索 |

**一句话结论**与摘要互补：摘要答「谁做的、多少钱档、能否平替、风险」；一句话答「好不好、什么档次」。

## 品类 → 国内公版平替检索

门禁通过后，按产品品类 **WebSearch**（勿凭记忆）：

| 品类 | 默认检索词 | PM 话术要点 |
|------|------------|-------------|
| TWS/音箱 | `{品类} 杰理 公版 turnkey`、`中科蓝讯 TWS 方案` | 多源方案厂，MOQ 低，同质化高 |
| 4G 穿戴 | `移远 EC600 公版`、`广和通 穿戴 方案` | 模组+音频双芯片常见 |
| BLDC 力反馈/伺服 | `STM32 FOC 双电机 参考设计`、`OpenFFB`、`EG2133 BLDC` | **高端品牌通常无直接平替**；DIY 仅参考力控思路 |
| 充电宝 | `南芯 快充 公版`、`智融 SW 协议 方案` | 协议芯片决定兼容性 |
| 智能家居 | `乐鑫 ESP32 公版`、`博流 BLE 方案` | 模组化程度高 |
| 工业 EtherCAT | `EtherCAT IO 参考设计` | 常与品牌自研绑定 |

平替难度：

- **低**：公版丝印 + 标准 Layout，任意方案厂可接
- **中**：公版硬件 + 定制固件/结构
- **高**：自研 PCB 料号、双 MCU 分轴、力控算法绑定固件

## 成本粗估方法

**免责声明（必填）**：粗估非报价，不含 SMT、外壳、电机、包装、认证；实际受批量、渠道、关税影响。

### 步骤

1. **PCB 费**：层数 + 板面积 → 查 [reference.md](reference.md) 层数档位表
2. **关键 IC**：WebSearch `"{型号}" 价格 OR price OR BOM 拆解`；无数据写「档位感」
3. **被动件裕量**：关键 IC 合计 × **1.25–1.35**（阻容感、连接器）— **仅计入中+置信度的 IC**
4. **输出宽区间**：取整到「十元」或「百元」档，写明「贵在哪」

### 置信度门禁（必填）

成本粗估的**每一项**（PCB 层数、单颗关键 IC、合计）必须绑定与元器件识别**同源**的置信度；**置信度低于「中」时，不得对该项做价格推测**。

**单项规则**

- 关键 IC 型号置信度 **低**（仅 Logo、Layout、图像描述）→ 该行写：`暂不可估（待确认 {型号}）`，**禁止**写 ¥ 区间
- PCB 层数置信度 **低** → 不写 PCB 费数字，写「层数待确认，PCB 费暂不可估」
- 仅当型号 **中/高** 或 WebSearch 有明确拆解/BOM 来源时，才可写单价感

**合计规则**

- **禁止**把「低置信度占位项」代入 ×1.3 公式
- 合计前置：至少 **主板上一颗最贵关键料**（主控/模组）为 **中+** 置信度，且 PCB 层数为 **中+**；否则整段写：
  > 主板 BOM 合计：**暂不可估**（关键料 {列表} 型号未确认，补拍微距后可粗估）

**PM 摘要中的成本档位**

- 有合计区间 → 写档位 + 区间
- 无合计 → 只写定性档位（入门/主流/中高端），**不写** ¥ 数字，并注明「BOM 待确认」

**低置信度时的成本输出示例**

```markdown
## 成本粗估
- 主板 PCB：层数推测四层（**低**）→ **暂不可估**，需背面/剖面确认
- 关键 IC：
  - 移远 EC600K-CN（**高**）→ ¥28–45
  - 音频 SoC 仅见 JL Logo（**低**）→ **暂不可估**，需型号行微距
- 主板 BOM 合计：**暂不可估**（1 项关键料未达中置信度）
- 成本结论：结构上最贵疑为 4G 模组，但整机 BOM 数字需补拍后给出
```

## 空贴 / 预留位推理

清点表状态为 **未贴装** 的位号，报告中 **必填** 推理段。

### 模式库

| 模式 | 可能含义 | 置信度 | Layout 依据 |
|------|----------|--------|-------------|
| 同封装邻位已贴、本位空 | 高低配 BOM（高配贴、低配不贴） | 中 | 焊盘有锡膏纹、丝印齐全 |
| 6-pin 空位靠近编码器/传感器座 | 第二路缓冲/驱动/电平转换 | 低–中 | 走线从传感器座分出 |
| 空位靠近通信连接器 | RS-422/RS-485/CAN 收发器预留 | 低–中 | 差分线对止于空焊盘 |
| 空位在电源入口旁 | 额外 LDO/保护/ORing | 低 | 电源树分支焊盘 |
| 测试点密集区空位 | 产测/校准/调试预留 | 低 | 邻近 TP 编号 |
| 无任何走线连接的空焊盘 | 结构占位或废弃位号 | 中 | 孤立焊盘 |

### 输出格式

```markdown
| 位号 | 状态 | 可能功能 | 置信度 | 依据 |
| IC5 | 未贴装 | 第二路 RS-422 线路驱动或 LED 驱动预留 | 低 | 6-pin 封装靠近 XS1 与 LED1，走线未完全看清 |
```

**禁止**把空贴写成「确认不需要」；一律用「可能预留」。

## 关键芯片可替代/供应链对标（按品类路由）

按板上 **实际品类** 必填其一（可填多表）。**禁止**把一切主控都写成 STM32 vs GD32；音频 SoC、蜂窝模组、WiFi IoT 各有独立对标表。

### 品类触发逻辑

```
if 蓝牙音频SoC → 填「音频主控对标」（杰理/恒玄/络达/蓝讯等）
else if 蜂窝模组 → 填「模组对标」（移远/广和通/有方等）
else if WiFi/BT IoT SoC → 填「连接芯片对标」（瑞昱/乐鑫/博通集成/翱捷等）
else if 通用MCU → 填「MCU对标」（按进口厂商选行，见下表）
else if 已是国产 → 填「同级国产竞品」（非进口替代）
else → 本节写 N/A + 原因
```

| 维度 | 写什么 |
|------|--------|
| 价格 | 同封装近似型号批发价差（%） |

### STM32 vs GD32（仅电机/通用 MCU 品类）

**STM32 优势（人话）**：ST 电机库成熟、参考设计多、国际客户认账快。  
**STM32 劣势**：同规格通常更贵；部分型号交期长（**须本次检索，勿写死「断供」**）。  
**GD32 优势**：价低、多型号 pin-to-pin 对标 STM32F；本土代理响应快。  
**GD32 劣势**：G4 级高集成电机外设覆盖弱于 STM32G4；FOC 例程与社区体量小；高端客户认证成本高。

### 供应链检索词（按品类）

- 音频 SoC：`杰理 JL7018 供货` / `恒玄 BES 交期`
- 蜂窝：`移远 EC600K 交期` / `广和通 Cat1 模组 价格`
- 通用 MCU：`STM32 交期 2025 OR 2026` / `GD32 替代 STM32 电机` / `{进口型号} 替代 APM32 OR AT32 OR CH32`

无确凿新闻时写：「未发现公开断供报道；交期与价格需向代理商确认（检索日期：{YYYY-MM-DD}）」。

## 供应链风险速览

对以下 **进口或紧缺料** 各搜一次：

1. 主控 MCU
2. 射频/蜂窝模组
3. 专用传感器（iC-Haus、TDK、Bosch 等）
4. 功率器件（若品牌敏感）

## 写作禁忌（PM 层）

- 方案商 ≠ 芯片原厂（例：方案商 Brunner，编码器芯片 iC-Haus）
- 成本只给区间也必须解释「为什么贵/便宜」
- 成本单项置信度低于「中」→ **不写 ¥**，写「暂不可估」
- 音频 SoC **禁止** 用 GD32/STM32 对标表替代
- 空贴必须写「可能」，不写「没有功能」
- 供应链结论必须标注检索日期
- 术语后跟人话（例：FOC = 让电机有力矩感的控制算法）
